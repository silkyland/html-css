## 7. Design decisions

### D1: ใช้ `docs/` subdirectory
- **Decision:** VitePress ใน `docs/`, ไฟล์เดิมอยู่ root
- **Rationale:** VitePress แนะนำ `./docs` สำหรับ standalone site

### D2: LivePreview = Vue component + iframe
- **Decision:** Component รับ `html`+`css` props, render ใน iframe ผ่าน `srcdoc`
- **Rationale:** iframe แยก CSS ตัวอย่างจาก VitePress theme, เห็นผลจริง
- **Rejected:** `::: raw` ไม่แยก CSS จาก VitePress

### D3: แยกไฟล์ตามหมวดใหญ่
- **Decision:** `intro/`, `html/`, `css/`, `git/`, `workshop/` — รวมส่วนสั้นๆ
- **Rationale:** สมดุลระหว่างความละเอียดและการนำทาง

### D4: "ภาพประกอบ" → `::: details` collapsible
- **Decision:** แปลงเป็น collapsible container
- **Rationale:** เก็บคำแนะนำผู้สอน ไม่รบกวนนักเรียน

### D5: `>` blockquote → `::: tip` containers
- **Decision:** แปลงทั้งหมด
- **Rationale:** VitePress container สวยกว่า blockquote

### D6: CSS Grid หน้าเดียว `css/grid.md`
- **Decision:** หน้าเดียว
- **Rationale:** เนื้อหา Grid พื้นฐานพอเหมาะ 1 หน้า

### D7: LivePreview register globally
- **Decision:** Global registration ใน theme/index.ts
- **Rationale:** ใช้ในทุกหน้า ไม่ต้อง import ซ้ำ
