## 4. Audit findings

- **[SECURITY / DEFENSE-IN-DEPTH] Preview iframe ไม่มี sandbox** — `srcdoc` มาจาก lesson HTML/CSS (`docs/.vitepress/components/LivePreview.vue:27-52`) แต่ iframe ที่ `:69-76` ไม่มี `sandbox`; หากตัวอย่างในอนาคตมี script จะรันใน origin ของ site และเข้าถึง parent ได้ นอกจากนี้ iframe ไม่มี `title` สำหรับ assistive technology
- **[CORRECTNESS] เส้นทาง Git แยกจากชิ้นงาน** — `docs/git/init.md:7-13` สร้าง `my-shop-profile` ใหม่ที่ว่าง แต่ `docs/git/add.md:13-18` สั่ง add `index.html`/`style.css` โดยไม่เคยย้ายหรือสร้างไฟล์ในโฟลเดอร์นั้น ผู้เรียนทำตามตรง ๆ จะ stage ไม่สำเร็จ
- **[CORRECTNESS] โค้ดที่โชว์ไม่ตรงกับ preview** — `docs/html/links-images.md:13-23` แสดง path ที่ไม่มีจริง (`images/eco-bag.jpg`, `contact.html`) แต่ preview แอบแทนภาพด้วย data URI ที่ `:24-30`; ผู้เรียน copy โค้ดจะได้ broken image/route ต่างจากผลบนจอ
- **[CORRECTNESS] อธิบาย doctype เป็น HTML5 แบบหยุดนิ่ง** — `docs/html/structure.md:34-42` ระบุว่า doctype บอกว่าเป็น “HTML5” ทั้งที่ WHATWG ระบุ current language simply as HTML และมาตรฐานเป็น Living Standard (section 3)
- **[CORRECTNESS] สัญญา hands-on ไม่ตรงกับกิจกรรม** — `docs/intro/agenda.md:18-20` บอกทุกหัวข้อมีการลงมือ แต่ checkpoint ก่อน capstone พบเพียง `docs/html/structure.md:44-46` และ `docs/css/responsive.md:87-89`; ไม่มี task/expected result สำหรับ tags, semantics, selectors, box model, Flexbox หรือ Git milestones
- **[ACCESSIBILITY] เนื้อหาตัวอย่างสอนไม่ครบสิ่งที่ rubric ควรตรวจ** — ตารางไม่มี `caption`/header scope (`docs/html/lists-tables.md:18-28`), บทภาพสอน alt แบบเดียวโดยไม่แยก informative/decorative (`docs/html/links-images.md:38-43`), และ capstone rubric ไม่ตรวจ heading order, label association, keyboard/focus หรือ contrast (`docs/workshop/project.md:33-38`)
- **[RELIABILITY] หลักสูตร core แน่นเกินเวลาที่ประกาศ** — agenda ให้ HTML 60 นาทีและ CSS 75 นาที (`docs/intro/agenda.md:10-13`) แต่ nav บังคับผ่าน HTML 6 หน้าและ CSS 7 หน้า รวม Form, Flexbox, Grid, responsive (`docs/.vitepress/config.ts:21-41`); ไม่มีป้าย core/optional
- **[MAINTAINABILITY] มีแหล่งเนื้อหาสองชุดที่ drift แล้ว** — root document ไม่มี Grid แต่ rendered docs และ capstone มี Grid (`docs/css/grid.md:5-153`, `docs/workshop/project.md:19-21`); build ไม่มีการอ่าน root document (`package.json:5-9`)
- **[MAINTAINABILITY] เอกสารความรู้ชี้นำผิด** — `AGENTS.md:5-26` และ `docs/ARCHITECTURE.md:5-12` ปฏิเสธการมี Git/build/docs; agent ถัดไปจะใช้คำสั่งและสถาปัตยกรรมผิด
- **[MAINTAINABILITY] ภาพประกอบยังเป็น TODO ที่ผู้เรียนมองไม่เห็น** — collapsible placeholders เช่น `docs/intro/overview.md:30-32` และ `docs/workshop/project.md:40-42` ไม่ใช่สื่อการสอนจริง

## 7. Design decisions

### Decision 1: ทำ `LivePreview` เป็นขอบเขตปลอดภัยและ truthful
- **Options considered:** A) คง iframe เดิมและเตือนผู้เขียน; B) ใส่ `sandbox`, `title` และบังคับ preview/code ใช้ asset เดียวกัน
- **Decision:** เลือก B ใน `docs/.vitepress/components/LivePreview.vue` และแก้ `docs/html/links-images.md` ให้มี asset จริงใน `docs/public/images/workshop/`
- **Rationale:** ลด latent script privilege และทำให้สิ่งที่เห็นตรงกับสิ่งที่ copy โดยไม่เปลี่ยน framework
- **Rejected because:** A อาศัยวินัยผู้เขียนและยังคงผลลัพธ์หลอกผู้เรียน

### Decision 2: เพิ่มบท “ทำไม Web, HTML และ CSS จึงเกิดขึ้น” ก่อน syntax
- **Options considered:** A) กล่อง timeline สั้นใน `structure.md`; B) บท narrative แยกที่เชื่อม CERN → HTML → CSS → standards today
- **Decision:** เลือก B สร้าง `docs/intro/web-history.md` ความยาวสอน 15–20 นาที พร้อม timeline, first-site link, คำถามสะท้อน และ source notes
- **Rationale:** ผู้ใช้ขอประวัติเป็นแกนหลัก และแหล่งใน section 3 ทำให้เชื่อม “เหตุใด HTML เน้นความหมาย” กับ “เหตุใด CSS แยกรูปลักษณ์” ได้ตรงกว่า trivia
- **Rejected because:** A สั้นเกินกว่าจะอธิบายปัญหาเดิม บุคคล องค์กร และ Living Standard โดยไม่กลายเป็นรายชื่อปี

### Decision 3: ใช้โปรเจกต์เดียวสะสมจากไฟล์แรกถึง push
- **Options considered:** A) เก็บตัวอย่างหลายธุรกิจแยกบท; B) ใช้ “ร้านกาแฟชุมชน” ชุดเดียวและให้ทุกบทแก้ `my-shop-profile/index.html`/`style.css`
- **Decision:** เลือก B; ทุกบทหลักมี `เริ่มจาก`, `ลงมือ`, `ผลที่ควรเห็น`, `checkpoint` และ Git milestone หลังเริ่ม Git
- **Rationale:** แก้ write path โดยตรงและลด cognitive load; Git official path รองรับ init ใน codebase ที่มีอยู่แล้ว (section 3)
- **Rejected because:** A ให้ความหลากหลายแต่ทำให้ผู้เริ่มต้นต้องประกอบ snippets เองก่อน capstone

### Decision 4: ทำ curriculum contract และ instructor path ให้ชัด
- **Options considered:** A) ใส่ prerequisites ใน overview หน้าเดียว; B) เพิ่ม setup/troubleshooting/instructor checklist พร้อม lesson metadata สม่ำเสมอ
- **Decision:** เลือก B: สร้าง `docs/intro/setup.md`, `docs/workshop/troubleshooting.md`, `docs/workshop/instructor-checklist.md` และกำหนด template บทเรียนใน `docs/CONTRIBUTING.md`
- **Rationale:** ผู้เรียนและผู้สอนไม่ต้องเดา editor/browser/terminal/folder/expected state และผู้เขียนรักษารูปแบบเดียวกันได้
- **Rejected because:** A ไม่ครอบคลุม recovery ระหว่าง Git/auth/path errors และไม่มี contract สำหรับบทในอนาคต

### Decision 5: ผนวก accessibility และ validation ในงานจริง
- **Options considered:** A) สร้างบท accessibility ท้ายเล่ม; B) ใส่เกณฑ์ในบทที่เกี่ยวข้องและมี checkpoint รวมก่อนส่ง
- **Decision:** เลือก B พร้อมหน้า `docs/html/accessibility-validation.md` สำหรับสรุป/ตรวจ: lang, headings, landmarks, alt decision, form labels, table caption/scope, keyboard focus, contrast และ W3C validation
- **Rationale:** WAI ระบุว่า semantics และ text alternatives เป็นส่วนของโครงสร้างที่เข้าถึงได้ ไม่ใช่ของตกแต่งภายหลัง (section 3)
- **Rejected because:** A ทำให้ผู้เรียนสร้าง markup ผิดก่อนแล้วค่อยย้อนแก้ และ accessibility ดูเป็น optional

### Decision 6: แยก core กับ enrichment ตามเวลาจริง
- **Options considered:** A) ลดจำนวนหน้าโดยลบ Form/Grid/clone; B) เก็บหน้าแต่ติดป้าย `บทเสริม` และเอาออกจาก core agenda/rubric
- **Decision:** เลือก B: core = history, HTML structure/content/semantics/accessibility, CSS syntax/box/visual/Flexbox/responsive, Git init/add/commit/push; enrichment = tables/forms/Grid/clone
- **Rationale:** รักษาเนื้อหาที่มีค่าไว้โดยทำให้เส้นทาง 09:00–16:00 ทำได้จริง
- **Rejected because:** A ลด overload ได้แต่สูญเสีย reference ที่สร้างไว้แล้วโดยไม่จำเป็น

### Decision 7: ให้ `docs/` เป็น source of truth เดียว
- **Options considered:** A) เขียน generator รวม docs กลับเป็น root monolith; B) archive monolith และแก้เฉพาะ VitePress Markdown
- **Decision:** เลือก B: ย้าย `workshop-html-css-git.md` ไป `archive/workshop-html-css-git-v1.md`, ระบุว่า frozen legacy และไม่ใช้สอน; `docs/` เป็น canonical curriculum
- **Rationale:** VitePress คือระบบที่ build/render จริง และมีเนื้อหา Grid/LivePreview ที่ monolith ไม่มี; generator เพิ่มระบบสองทิศทางโดยไม่มีความจำเป็น
- **Rejected because:** A เพิ่ม maintenance surface และต้องกำหนด serialization ของ Vue components/containers กลับเป็น Markdown เดี่ยว

### Decision 8: ซ่อม project knowledge หลังโครงสร้างนิ่ง
- **Options considered:** A) แก้ AGENTS/ARCHITECTURE ด้วยมือระหว่าง rewrite; B) rewrite เสร็จแล้วใช้ deep-plan-ingest เพื่ออัปเดต durable knowledge จากหลักฐานจริง
- **Decision:** เลือก B และทำหลัง verification ผ่าน
- **Rationale:** ป้องกัน knowledge files บันทึกโครงสร้างชั่วคราว; เป็น workflow คู่กับ deep-plan
- **Rejected because:** A มีโอกาส stale ซ้ำระหว่างย้าย/เพิ่มบท

### Decision 9: เพิ่ม content checks แบบ dependency-free
- **Options considered:** A) พึ่ง VitePress build และ manual review; B) เพิ่ม Node script ตรวจ routes/assets/lesson blocks/canonical-source references
- **Decision:** เลือก B: สร้าง `scripts/check-curriculum.mjs` และ `npm run docs:check` โดยใช้ Node stdlib แล้วรันก่อน build
- **Rationale:** build ตรวจ syntax/render แต่ไม่พิสูจน์ว่าลิงก์ภายใน, asset หรือ checkpoint ที่หลักสูตรบังคับมีจริง
- **Rejected because:** A ปล่อย regression เชิงเนื้อหาผ่านได้แม้ site build สำเร็จ
