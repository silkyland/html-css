# กำหนดการหนึ่งวัน

เส้นทางหลักใช้เวลา **09:00–16:00** รวมเวลาพัก เนื้อหาบทเสริมไม่รวมในเวลาหลัก และไม่ใช้เป็นเกณฑ์ตรวจชิ้นงานท้ายวัน

## เส้นทางหลัก (Core)

| เวลา | กิจกรรมหลัก | Checkpoint |
|---|---|---|
| 09:00–09:20 | [เตรียมเครื่องและโปรเจกต์](/intro/setup) | เปิด `index.html` จากโฟลเดอร์ถูกต้อง |
| 09:20–09:40 | [จาก CERN สู่ Web ที่เราใช้วันนี้](/intro/web-history) | ตอบได้ว่า HTML และ CSS เกิดมาเพื่ออะไร |
| 09:40–10:00 | [Browser และ Server ทำงานร่วมกัน](/intro/how-web-works) | แยก request, response, HTML และ CSS ได้ |
| 10:00–10:45 | HTML: โครงสร้างและเนื้อหา | หน้าเว็บมีหัวเรื่อง ข้อมูลร้าน และเมนู |
| 10:45–11:00 | **พัก 15 นาที** | — |
| 11:00–11:45 | HTML: ลิงก์ รูปภาพ และ Semantic HTML | ใช้คีย์บอร์ดไปยังแต่ละส่วนได้ |
| 11:45–12:00 | Accessibility และ Validation | ตรวจรายการสำคัญและแก้ validation error |
| 12:00–13:00 | **พักกลางวัน 60 นาที** | — |
| 13:00–14:15 | CSS: สี กล่อง การจัดวาง และ Responsive | หน้าไม่ล้นที่ความกว้าง 320px |
| 14:15–14:30 | **พัก 15 นาที** | — |
| 14:30–15:20 | Git และ GitHub | baseline 1 commit ถูก push ขึ้น GitHub |
| 15:20–15:50 | ปรับแบรนด์และตรวจชิ้นงาน | มี 3 commit จากงานจริง, push รอบสุดท้าย และผ่าน rubric |
| 15:50–16:00 | สรุปและถาม–ตอบ | เปิด repository และอธิบายงานตัวเองได้ |

## จังหวะของวันแบบย่อ

- **เช้า:** เตรียมเครื่อง → เข้าใจที่มาของ Web → สร้างเนื้อหาด้วย HTML
- **บ่าย:** ตกแต่งด้วย CSS → บันทึกและส่งงานด้วย Git/GitHub → ปรับแบรนด์และตรวจชิ้นงาน

ถ้าหลงทาง ให้กลับมาดูตารางนี้ก่อนถามว่า “ตอนนี้ควรอยู่บทไหน”

- [รายการและตาราง](/html/lists-tables)
- [ฟอร์มติดต่อ](/html/forms)
- CSS Grid
- การ clone repository

ถ้าผลบนหน้าจอไม่ตรงกับบท ให้หยุดแก้ก่อนเริ่มหัวข้อถัดไป เพราะทุกบทต่อยอดไฟล์ชุดเดิมใน `my-shop-profile`
:::
