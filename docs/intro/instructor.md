# รู้จักวิทยากร

## นายบัณฑิต นันทะเทศ

![นายบัณฑิต นันทะเทศ กำลังบรรยายในห้องปฏิบัติการคอมพิวเตอร์](/images/workshop/instructor-bandhit.webp)

- Senior Full Stack Developer, Data Horizon Co., Ltd.
- ทำงานด้านการพัฒนาเว็บไซต์ทั้งส่วน Front-end (สิ่งที่ผู้ใช้เห็น) และ Back-end (ระบบหลังบ้าน)
- สนใจการเปลี่ยนโจทย์ทางธุรกิจให้เป็นระบบที่ใช้งานได้จริง และถ่ายทอดทักษะพื้นฐานให้ผู้เริ่มต้นลงมือได้เอง

วันนี้เป้าหมายไม่ใช่ให้จำ tag หรือคำสั่งทั้งหมด แต่ให้สร้างชิ้นงานหนึ่งชิ้นที่อธิบายและพัฒนาต่อได้หลังจบเวิร์กชอป

## วิธีเรียนร่วมกันวันนี้

ผู้เรียนไม่จำเป็นต้องจำทุกอย่าง สิ่งสำคัญคือฝึกสี่ทักษะนี้:

1. ค้นคำตอบจากเอกสารที่น่าเชื่อถือ เช่น เอกสารในหลักสูตรนี้ หรือแหล่งมาตรฐานที่อ้างไว้ท้ายบท
2. อ่านผลบน Browser และข้อความ error แล้วเชื่อมกับไฟล์ที่กำลังแก้
3. ย้อนกลับไปยัง checkpoint ล่าสุดเมื่อผลไม่ตรง แทนการข้ามไปบทถัดไป
4. อธิบายเหตุผลของโค้ดที่ตัวเองเลือกใช้ได้ด้วยภาษาของตนเอง

::: tip ขอความช่วยเหลือให้แก้ได้เร็ว
เมื่อเรียกผู้สอน ช่วยบอกสามอย่าง: **กำลังทำบทไหน**, **คาดว่าจะเห็นอะไร** และ **ตอนนี้เห็นอะไร** พร้อมเปิดไฟล์กับ Browser ไว้ข้างกัน
:::

## ข้อตกลงในห้อง

- ถามได้ทันทีเมื่อไม่เข้าใจ ไม่มีคำถามใดเล็กเกินไปสำหรับผู้เริ่มต้น
- ใช้ข้อมูลร้านและข้อมูลติดต่อสมมติในแบบฝึกหัด
- ไม่คัดลอก token, password หรือข้อมูลส่วนตัวลงใน repository
- ช่วยเพื่อนด้วยการอธิบายแนวคิด ไม่ทำงานแทนทั้งหมด
- บันทึกไฟล์บ่อย ๆ และดู path บนแถบที่อยู่ของ Browser ให้แน่ใจว่าเปิดไฟล์ชุดที่กำลังแก้จริง
