# บทเสริม: รายการและตารางข้อมูล

**สถานะ: บทเสริม (Enrichment) · เวลาแนะนำ 20 นาที**

บทนี้ไม่ใช่เงื่อนไขของชิ้นงานหลัก เหมาะสำหรับผู้เรียนที่ผ่าน checkpoint Accessibility แล้ว

## เป้าหมาย

เลือก `<ul>`/`<ol>` ตามความสำคัญของลำดับ และสร้างตารางเวลาชงกาแฟที่มี caption กับหัวตารางชัดเจน

## สถานะเริ่มต้น

ใช้ `index.html` ร้านกาแฟบ้านดอยฉบับ semantic จากบทหลัก

## รายการ: ลำดับสำคัญหรือไม่?

```html
<h2>บริการของเรา</h2>
<ul>
  <li>เมล็ดกาแฟคั่ว</li>
  <li>เครื่องดื่มหน้าร้าน</li>
  <li>เวิร์กชอปดริปกาแฟ</li>
</ul>

<h2>วิธีสั่งเมล็ดกาแฟ</h2>
<ol>
  <li>เลือกชนิดและระดับการคั่ว</li>
  <li>แจ้งรูปแบบการบด</li>
  <li>ยืนยันที่อยู่และชำระเงิน</li>
</ol>
```

ใช้ `<ol>` เมื่อการสลับลำดับทำให้ความหมายหรือผลลัพธ์เปลี่ยน ใช้ `<ul>` เมื่อเป็นกลุ่มรายการที่ไม่มีลำดับตายตัว

## ลงมือ

### 1. เพิ่มรายการบริการและขั้นตอนสั่งซื้อ

ใน `index.html` เพิ่มส่วนรายการลงใน `<main>` (หรือส่วนที่เหมาะสม) ตามตัวอย่างด้านบน แล้วเปิด Browser ดูจุดกับตัวเลข

### 2. (ถ้าเหลือเวลา) เพิ่ม description list

ลองเพิ่ม `<dl>` สำหรับศัพท์กาแฟตามตัวอย่างด้านล่าง — ไม่บังคับใน checkpoint

## รายการคำอธิบาย: `<dl>`, `<dt>`, `<dd>`

นอกจาก `<ul>` และ `<ol>` ยังมี `<dl>` (description list) สำหรับคู่คำ-คำอธิบาย เช่นศัพท์กับนิยาม หรือชื่อเมนูกับรายละเอียด:

```html
<h2>ศัพท์กาแฟที่ควรรู้</h2>
<dl>
  <dt>อเมริกาโน่</dt>
  <dd>เอสเปรสโซผสมน้ำร้อน ได้รสกาแฟที่เบากว่าลาเต้</dd>

  <dt>ดริป</dt>
  <dd>วิธีชงที่เทน้ำร้อนผ่านเมล็ดกาแฟบดในกระดาษกรองทีละส่วน</dd>
</dl>
```

| Tag | หน้าที่ |
|---|---|
| `<dl>` | ครอบคู่คำ-คำอธิบายทั้งหมด |
| `<dt>` | คำที่ต้องการอธิบาย |
| `<dd>` | คำอธิบายของ `<dt>` ด้านบน |

Browser เยื้อง `<dd>` เข้าด้านขวาโดยค่าเริ่มต้น ทำให้เห็นความสัมพันธ์ระหว่างคำและคำอธิบายโดยไม่ต้องใช้ CSS

## ตาราง: ใช้กับข้อมูล ไม่ใช่จัดหน้า

### 3. เพิ่มตารางรอบเวิร์กชอป

เพิ่มใน `<main>`:

```html
<section aria-labelledby="workshop-title">
  <h2 id="workshop-title">รอบเวิร์กชอปดริปกาแฟ</h2>
  <table>
    <caption>ตารางรอบวันเสาร์ ราคา 350 บาทต่อคน</caption>
    <thead>
      <tr>
        <th scope="col">รอบ</th>
        <th scope="col">เวลา</th>
        <th scope="col">จำนวนที่ว่าง</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <th scope="row">รอบเช้า</th>
        <td>09:30–11:00</td>
        <td>4 คน</td>
      </tr>
      <tr>
        <th scope="row">รอบบ่าย</th>
        <td>13:30–15:00</td>
        <td>2 คน</td>
      </tr>
    </tbody>
  </table>
</section>
```

`caption` บอกว่าตารางนี้เกี่ยวกับอะไร ส่วน `scope` เชื่อมหัวตารางกับข้อมูลในแถวหรือคอลัมน์ ไม่ควรใช้ table เพื่อจัดตำแหน่งภาพหรือกล่องสินค้า

## ผลที่ควรเห็น

Browser แสดงรายการแบบจุด รายการแบบตัวเลข และตารางสองแถว แม้ยังไม่สวยแต่ความสัมพันธ์ของข้อมูลอยู่ใน HTML แล้ว

## จุดที่พลาดบ่อย

- ใช้ `<table>` เพื่อจัดวางภาพหรือกล่องสินค้าให้อยู่ในแนวเดียวกัน ทั้งที่ไม่ใช่ข้อมูลตาราง
- ลืม `<caption>` ทำให้ผู้ใช้ screen reader ไม่รู้ว่าตารางนี้เกี่ยวกับอะไรก่อนเข้าไปอ่านทีละเซลล์
- ใส่ `scope="col"` ให้หัวแถว หรือ `scope="row"` ให้หัวคอลัมน์ สลับกัน ทำให้การเชื่อมหัวตารางกับข้อมูลผิด
- ใช้ `<ul>` ทั้งที่ลำดับข้อมูลมีความหมาย เช่น ขั้นตอนการสั่งซื้อที่ต้องทำตามลำดับ

## Checkpoint

- [ ] ขั้นตอนที่ต้องเรียงลำดับใช้ `<ol>`
- [ ] ตารางมี `<caption>` ที่เข้าใจได้โดยไม่ต้องอ่านย่อหน้ารอบข้าง
- [ ] หัวคอลัมน์ใช้ `scope="col"` และหัวแถวใช้ `scope="row"`
- [ ] ไม่มี table ที่สร้างขึ้นเพื่อจัด layout
