# เติมเนื้อหาด้วย Tag ที่มีความหมาย

**สถานะ: บทหลัก (Core) · เวลาแนะนำ 15 นาที**

## เป้าหมาย

เพิ่มหัวเรื่อง ย่อหน้า ข้อความสำคัญ และรายการเมนู โดยเลือกระดับ heading จากโครงสร้าง ไม่ใช่จากขนาดตัวอักษร

## สถานะเริ่มต้น

ใช้ `index.html` จากบทโครงสร้าง ภายใน `<body>` มีชื่อร้านและคำแนะนำสั้น ๆ

## หลักคิด: ความหมายมาก่อนหน้าตา

- `<h1>` คือหัวเรื่องหลักของหน้า บทนี้ใช้หนึ่งครั้ง
- `<h2>` คือหัวเรื่องของส่วนสำคัญใต้ `<h1>`
- `<p>` คือย่อหน้า ไม่ใช่เครื่องมือเว้นระยะ
- `<strong>` บอกว่าข้อความมีความสำคัญ
- `<ul>` และ `<li>` ใช้เมื่อข้อมูลเป็นรายการที่ไม่ต้องเรียงลำดับ

## ลงมือ

### ขยายเนื้อหาใน `<body>`

```html
<body>
  <h1>กาแฟบ้านดอย</h1>
  <p>กาแฟจากเกษตรกรในชุมชน คั่วสดทุกสัปดาห์</p>

  <h2>เมนูแนะนำ</h2>
  <ul>
    <li>อเมริกาโน่น้ำผึ้ง — 65 บาท</li>
    <li>ลาเต้นมสด — 70 บาท</li>
    <li>ชาดอกกาแฟ — 55 บาท</li>
  </ul>

  <h2>เวลาเปิดร้าน</h2>
  <p>เปิดทุกวัน <strong>08:00–17:00 น.</strong></p>
</body>
```

## ลำดับ heading ที่อ่านเป็นโครงร่างได้

ลองอ่านเฉพาะ heading:

```text
h1 กาแฟบ้านดอย
├── h2 เมนูแนะนำ
└── h2 เวลาเปิดร้าน
```

อย่าเลือก `<h4>` เพียงเพราะอยากได้ตัวอักษรเล็กกว่า เราจะควบคุมขนาดด้วย CSS ในช่วงบ่าย

## ผลที่ควรเห็น

หน้าเว็บมีหัวเรื่องหลักหนึ่งรายการ หัวเรื่องระดับสองสองรายการ และรายการเมนูสามรายการ เนื้อหายังดูเรียบง่ายเพราะยังไม่ได้ใช้ CSS

## Checkpoint

- [ ] หน้าเว็บมี `<h1>` หนึ่งตัว
- [ ] หัวข้อย่อยใช้ `<h2>` และไม่ข้ามระดับโดยไม่มีเหตุผล
- [ ] เมนูทั้งสามอยู่ใน `<ul>` เดียวกัน
- [ ] ไม่มี `<br>` หลายตัวเพื่อสร้างช่องว่าง

## บันทึกแหล่งข้อมูล

การใช้ heading เพื่อสื่อโครงสร้างและช่วยการนำทางอ้างจาก [W3C WAI — Headings](https://www.w3.org/WAI/tutorials/page-structure/headings/)
