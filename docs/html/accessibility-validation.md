# ตรวจการเข้าถึงและความถูกต้องก่อนตกแต่ง

**สถานะ: บทหลัก (Core) · เวลาแนะนำ 15 นาที**

## เป้าหมาย

ตรวจหน้าเว็บด้วยคีย์บอร์ด ตรวจ semantics ที่สำคัญ และใช้ W3C Validator หา markup ที่ผิดก่อนเริ่ม CSS

## สถานะเริ่มต้น

`index.html` มี semantic structure ลิงก์ในหน้า และภาพ local จากบทก่อนหน้า บันทึกไฟล์ล่าสุดก่อนตรวจ

## Accessibility ไม่ใช่ “โหมดพิเศษ”

ผู้ใช้บางคนใช้คีย์บอร์ด บางคนขยายจอ บางคนใช้ screen reader และทุกคนอาจเจอสถานการณ์ที่มองจอหรือใช้เมาส์ไม่สะดวก HTML ที่มีความหมายช่วยให้เครื่องมือเหล่านี้เข้าใจหน้าได้ตั้งแต่ต้น

## ลงมือ

### 1. Keyboard pass

1. คลิกแถบที่อยู่ แล้วกด `Tab` เพื่อเข้าหน้า
2. กด `Tab` ต่อไปโดยไม่ใช้เมาส์
3. ตรวจว่าลิงก์มีลำดับสมเหตุผลและเห็นว่าตอนนี้ focus อยู่ตรงไหน
4. กด `Enter` ที่ลิงก์ในเมนู ตรวจว่าไปยังส่วนถูกต้อง

> รูปลักษณ์ focus จะชัดขึ้นในบท CSS แต่ห้ามลบ outline โดยไม่มีสิ่งทดแทน

### 2. ตรวจโครงสร้างด้วย checklist

- `<html lang="th">` ตรงกับภาษาหลัก
- มี `<title>` ที่บอกชื่อหน้า
- มี `<h1>` หนึ่งหัวเรื่องหลัก และหัวเรื่องรองเรียงอย่างมีเหตุผล
- มี `header`, `nav`, `main` และ `footer` ตามหน้าที่
- ภาพให้ข้อมูลมี `alt`; ภาพตกแต่งใช้ `alt=""`
- ข้อความลิงก์อธิบายปลายทางได้ หลีกเลี่ยงคำว่า “คลิกที่นี่” เพียงอย่างเดียว
- เมื่อมีฟอร์ม ทุกช่องกรอกมี `<label>` ที่เชื่อมด้วย `for`/`id`
- เมื่อมีตารางข้อมูล มี `<caption>` และ `<th scope="col">` หรือ `scope="row"` ตามโครงสร้าง

### 3. ตรวจด้วย W3C Markup Validation Service

1. เปิด [validator.w3.org](https://validator.w3.org/)
2. เลือกแท็บ **Validate by Direct Input**
3. คัดลอก `index.html` ทั้งไฟล์ไปวาง
4. กด **Check**
5. แก้รายการที่เป็น Error ทีละข้อจากบนลงล่าง แล้วตรวจซ้ำ

Validator ช่วยหาปัญหา syntax เช่น tag ปิดผิดตำแหน่ง, attribute ซ้ำ หรือ element อยู่ผิดบริบท แต่ไม่ได้รับรองว่าข้อความเข้าใจง่าย ดีไซน์อ่านได้ หรือ `alt` สื่อความหมายถูก คุณยังต้องตรวจด้วยคน

### ทดลองอ่าน Error

ลองลบ `>` จาก tag ปิดหนึ่งตัวชั่วคราว ตรวจอีกครั้ง แล้วอ่านเลขบรรทัดที่ Validator รายงาน จากนั้นกด undo และตรวจจนไม่เหลือ error ที่เกิดจากโค้ดบทเรียน

## ผลที่ควรเห็น

- ใช้ `Tab` เข้าถึงลิงก์ทุกตัวและกดใช้งานได้
- HTML Validator ไม่รายงาน error จาก markup ของบทเรียน
- checklist ข้างต้นผ่านทุกข้อที่มีอยู่ในหน้า

## Checkpoint

ก่อนเริ่ม CSS ตรวจให้ครบ:

- [ ] Keyboard pass ผ่าน
- [ ] ลำดับ landmarks และ headings สมเหตุผล
- [ ] ภาพทุกภาพผ่านการตัดสินใจเรื่อง `alt`
- [ ] Validator ไม่มี error ที่ยังไม่ได้อธิบาย
- [ ] เก็บ `index.html` นี้เป็นสถานะตั้งต้นของบท CSS

## บันทึกแหล่งข้อมูล

หลักการ page structure และ images อ้างจาก [W3C WAI — Page Structure](https://www.w3.org/WAI/tutorials/page-structure/) และ [W3C WAI — Images](https://www.w3.org/WAI/tutorials/images/) เครื่องมือตรวจ markup คือ [W3C Markup Validation Service](https://validator.w3.org/)
