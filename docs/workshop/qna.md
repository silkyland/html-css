# ทบทวนและถาม–ตอบ

ก่อนถาม ลองจัดคำถามให้มีหลักฐาน เพื่อให้ผู้สอนช่วยได้ตรงจุด:

```text
ฉันกำลังทำ:
คาดว่าจะเห็น:
แต่เห็นจริงว่า:
คำสั่ง/ไฟล์ที่เกี่ยวข้อง:
สิ่งที่ลองแล้ว:
```

## คำถามสะท้อนการเรียนรู้

- ส่วนใดเปลี่ยน mental model ของคุณมากที่สุด: ประวัติ Web, semantic HTML, responsive CSS หรือ Git?
- checkpoint ใดจับข้อผิดพลาดได้ก่อนส่งงาน?
- ถ้าทำเว็บไซต์ถัดไป คุณจะวาง milestone commit อย่างไร?
- มีจุดใดใน rubric ที่ยังแสดงหลักฐานไม่ได้?

หากยังมี error ให้เปิด [คู่มือแก้ปัญหา](/workshop/troubleshooting) แล้วนำ output จริงมาถาม หากงานผ่านแล้วให้เลือกเส้นทางจาก [สรุป](/workshop/summary)
