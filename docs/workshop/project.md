# Capstone: เปลี่ยน Starter ให้เป็นแบรนด์ของคุณ

> **บทเรียนหลัก (Core)** — ใช้ไฟล์ `my-shop-profile/index.html` และ `style.css` ชุดเดิม ไม่เริ่มจากการคัดลอกคำตอบสำเร็จรูป

## เป้าหมาย

สร้างเว็บไซต์หนึ่งหน้าสำหรับธุรกิจสมมติหรือกิจการจริง เปลี่ยนชื่อ เนื้อหา สี และภาพจาก “ร้านกาแฟชุมชน” ให้เป็นแบรนด์ของคุณ พร้อมหลักฐานที่ตรวจซ้ำได้ทั้ง HTML, CSS และ Git

## เริ่มจาก

ก่อนเริ่มงานตรวจว่าอยู่ในโปรเจกต์เดิมและมีประวัติ:

```bash
pwd
git status
git log --oneline -3
```

เลือกร้านกาแฟ เบเกอรี สินค้าชุมชน ร้านเสื้อผ้า หรือบริการหนึ่งประเภท แล้วเขียนสรุปสั้น ๆ ก่อนลงโค้ด:

```text
ชื่อแบรนด์:
ลูกค้าหลัก:
คำโปรยหนึ่งประโยค:
สินค้า/บริการ 3 รายการ:
ช่องทางติดต่อ:
```

## ลงมือทำ

1. เปลี่ยนข้อความทุกส่วนให้เป็นแบรนด์ของคุณ ไม่เหลือข้อความ placeholder
2. ใช้ `header`, `nav`, `main`, section เนื้อหา และ `footer` ตามความหมาย
3. ใส่หัวข้อเรียงลำดับ ภาพพร้อม alt ที่เหมาะสม และลิงก์ที่ใช้งานได้
4. ปรับสี typography spacing, card และ navigation ด้วย CSS
5. ทำ layout หลักด้วย Flexbox และปรับให้ใช้ได้ที่ 320, 768 และ 1280 พิกเซล
6. ตรวจ keyboard focus, horizontal overflow และ HTML Validator
7. ใช้ `git diff` ตรวจงานแล้วบันทึกเป็น milestone ที่มีความหมาย

CSS Grid เป็น [บทเสริม](/css/grid) ไม่ใช่ข้อบังคับของ capstone นี้

## หลักฐานส่งงาน

ส่ง URL ของ GitHub repository และเตรียมแสดง 4 อย่าง:

- หน้าเว็บที่เปิดจากไฟล์ของตนเอง
- ผล W3C HTML Validator ที่ไม่มี error จากโค้ดที่ส่ง
- หน้าจอที่ความกว้าง 320px โดยไม่มี horizontal scroll
- ผล `git log --oneline -3` และประวัติเดียวกันบน GitHub

หากอินเทอร์เน็ตล่ม ส่งโฟลเดอร์งานพร้อม output ของ `git log --oneline -3` ก่อน แล้ว push ภายหลัง

## Rubric แบบผ่าน/ไม่ผ่าน

แต่ละหมวด **ผ่าน** เมื่อผ่านทุกข้อในหมวด หากยังไม่ผ่าน ให้ใช้ evidence ระบุจุดที่ต้องแก้ ไม่ให้คะแนนจากความชอบส่วนตัว

### 1. HTML semantics และ accessibility

| เกณฑ์ผ่าน | Evidence ที่ผู้ตรวจใช้ |
|---|---|
| มี `lang="th"`, charset, viewport และ title ที่ตรงกับแบรนด์ | เปิด source ของ `index.html` |
| ใช้ `header`, `nav`, `main` และ `footer`; มี `h1` เพียงหัวข้อหลักและหัวข้อย่อยไม่ข้ามระดับโดยไม่มีเหตุผล | Inspect document outline/source |
| ภาพสื่อความหมายมี alt ที่บอกสาระ; ภาพตกแต่งใช้ `alt=""` | ตรวจ `<img>` ทุกตัว |
| ลิงก์เข้าถึงด้วย Tab ได้และ focus มองเห็น; ลิงก์ภายในชี้เป้าหมายที่มีจริง | ทดลอง keyboard ตั้งแต่ต้นหน้า |
| Validator ไม่มี HTML error ที่เกิดจากชิ้นงาน | URL หรือภาพผล validator |

### 2. CSS และ responsive

| เกณฑ์ผ่าน | Evidence ที่ผู้ตรวจใช้ |
|---|---|
| `style.css` เชื่อมกับ HTML และมีการออกแบบสี ตัวอักษร spacing, box model และ Flexbox | ปิด/เปิด stylesheet เทียบและอ่าน source |
| ข้อความอ่านได้ สีข้อความ/พื้นหลังไม่กลืนกัน และ focus indicator ชัด | ตรวจหน้าและเดินด้วย Tab |
| ภาพไม่ล้น container และหน้ากว้าง 320, 768, 1280px ไม่มี horizontal overflow | DevTools responsive mode 3 ขนาด |
| เนื้อหา/การ์ดเรียงใหม่อย่างเหมาะสมบนจอแคบ | เปรียบเทียบ 320px กับ desktop |

### 3. ประวัติ Git

| เกณฑ์ผ่าน | Evidence ที่ผู้ตรวจใช้ |
|---|---|
| มีอย่างน้อย 3 commit ที่เกิดจากการพัฒนาจริง | `git log --oneline --reverse` |
| ข้อความแต่ละ commit ระบุงานหนึ่งเรื่อง ไม่ใช้เพียง `update`, `test`, `final` | อ่าน commit messages |
| `git status` เป็น clean ตอนส่ง | output ของ `git status` |
| local `main` และ GitHub มี commit ล่าสุดเดียวกัน | เปรียบเทียบ hash ย่อใน terminal/GitHub |

### 4. ความครบถ้วนของชิ้นงาน

| เกณฑ์ผ่าน | Evidence ที่ผู้ตรวจใช้ |
|---|---|
| ชื่อแบรนด์ คำโปรย คำแนะนำ สินค้า/บริการอย่างน้อย 3 รายการ และช่องทางติดต่อครบ | เปิดหน้าเว็บตั้งแต่ต้นถึงท้าย |
| ไม่มี placeholder เช่น `USERNAME`, `Your Name`, lorem ipsum หรือลิงก์/ภาพเสีย | ค้น source และคลิกทุกลิงก์ |
| repository มี `index.html`, `style.css` และ asset ที่อ้างถึงครบ | เปิด file tree บน GitHub |

## ผลที่ควรเห็น

เว็บไซต์สื่อถึงแบรนด์ใหม่อย่างชัดเจน ใช้ keyboard และจอเล็กได้ เปิด asset ครบ และ repository แสดงพัฒนาการอย่างน้อย 3 milestone

## Checkpoint ก่อนส่ง

- [ ] ผ่านทุกข้อใน rubric ทั้ง 4 หมวด
- [ ] `git status` เป็น clean
- [ ] `git push` สำเร็จหรือบันทึกเหตุผล fallback ไว้
- [ ] เปิด URL repository ในหน้าต่าง private/incognito ได้ตามสิทธิ์ที่ตั้งใจ
