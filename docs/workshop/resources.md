# แหล่งเรียนรู้และแหล่งอ้างอิง

เลือกอ่านตามเป้าหมาย ไม่จำเป็นต้องอ่านมาตรฐานทั้งฉบับเพื่อเริ่มทำเว็บ แหล่งมาตรฐานใช้ยืนยันพฤติกรรม ส่วน tutorial ใช้ฝึกทีละขั้น

## แหล่งปฐมภูมิ: ประวัติ Web และมาตรฐาน

- [CERN: A short history of the Web](https://home.cern/science/computing/the-birth-of-the-web/short-history-web/) — ข้อเสนอปี 1989, browser/server แรก และการเปิด Web แบบ royalty-free
- [W3C: History](https://www.w3.org/about/history/) — บทบาทของ Tim Berners-Lee, CERN และการก่อตั้ง W3C
- [WHATWG: HTML Living Standard — Introduction](https://html.spec.whatwg.org/multipage/introduction.html) — เป้าหมายและวิวัฒนาการของ HTML ปัจจุบัน
- [W3C: A brief history of CSS](https://www.w3.org/Style/LieBos2e/history/) — ข้อเสนอของ Håkon Wium Lie และการพัฒนาร่วมกับ Bert Bos
- [W3C: CSS Snapshot](https://www.w3.org/TR/css/) — ภาพรวมโมดูล CSS ที่เป็นมาตรฐาน
- [W3C WAI: Page Structure](https://www.w3.org/WAI/tutorials/page-structure/) และ [Images](https://www.w3.org/WAI/tutorials/images/) — landmark, headings และ text alternative
- [W3C Markup Validation Service](https://validator.w3.org/) — ตรวจ HTML ด้วย URL, upload หรือ direct input

## เอกสารทางการของ Git และ GitHub

- [Pro Git: Getting Started](https://git-scm.com/book/en/v2/Getting-Started-About-Version-Control) — mental model ของ Version Control
- [Git 2.54: git-init](https://git-scm.com/docs/git-init), [git-add](https://git-scm.com/docs/git-add), [git-commit](https://git-scm.com/docs/git-commit) — จาก repository ถึง commit
- [Git 2.54: git-status](https://git-scm.com/docs/git-status), [git-diff](https://git-scm.com/docs/git-diff), [git-log](https://git-scm.com/docs/git-log) — ตรวจสถานะและหลักฐาน
- [Git 2.54: git-remote](https://git-scm.com/docs/git-remote), [git-push](https://git-scm.com/docs/git-push), [git-clone](https://git-scm.com/docs/git-clone) — local/remote workflow
- [GitHub Docs: Get started using Git](https://docs.github.com/en/get-started/using-git/about-git) — ภาพรวม Git ในบริบท GitHub
- [GitHub Docs: Authentication](https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/about-authentication-to-github) — วิธีพิสูจน์ตัวตนตามช่องทาง

## Tutorial สำหรับผู้เริ่มต้น

- [MDN Learn Web Development](https://developer.mozilla.org/en-US/docs/Learn_web_development) — เส้นทาง HTML/CSS ที่เน้นพื้นฐานและ accessibility
- [web.dev Learn HTML](https://web.dev/learn/html/) และ [Learn CSS](https://web.dev/learn/css/) — บทเรียนพร้อมหัวข้อย่อย
- [GitHub Skills](https://skills.github.com/) — แบบฝึก repository ผ่าน GitHub
- [Flexbox Froggy](https://flexboxfroggy.com/) — เกมฝึก main/cross axis
- [Grid Garden](https://cssgridgarden.com/) — เกมสำหรับบทเสริม CSS Grid

## แผนฝึก 4 สัปดาห์

1. **สัปดาห์ 1:** ทำหน้าแนะนำตัวใหม่โดยใช้ semantic HTML และ validator
2. **สัปดาห์ 2:** ทำ card layout ด้วย Flexbox แล้วทดสอบ 320/768/1280px
3. **สัปดาห์ 3:** ทำหน้าแบรนด์อีกประเภทโดยไม่ดูไฟล์เดิม และ commit ทุก milestone
4. **สัปดาห์ 4:** ให้เพื่อนทดสอบ keyboard/ลิงก์/จอเล็ก เปิด issue แล้วแก้พร้อม commit ที่อ้างถึงปัญหา

ทุกสัปดาห์ให้ตอบสามคำถาม: “ผู้ใช้ทำอะไรได้”, “ตรวจว่าผ่านอย่างไร” และ “commit ไหนเป็นหลักฐาน” วิธีนี้ทำให้ portfolio แสดงกระบวนการ ไม่ใช่เพียงภาพสุดท้าย
