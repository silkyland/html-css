# เช็กลิสต์สำหรับผู้สอน

> หน้านี้เป็นคู่มือดำเนินชั้นเรียน ใช้ร่วมกับ [กำหนดการ](/intro/agenda) และปรับเวลา checkpoint ให้ตรงกับวันจริง

## ก่อนวันสอน: Preflight

- [ ] เปิดทุก route จาก sidebar และรัน `npm test` จาก project root
- [ ] ทดลองขั้นตอนทั้งหมดด้วยบัญชี/โฟลเดอร์ learner ใหม่ ไม่ใช้ credential หรือ cache ของผู้สอน
- [ ] ทดสอบ Chrome/Firefox, VS Code, Terminal, `git --version` และการเปิด local HTML บนเครื่องห้อง
- [ ] ให้ผู้เรียนสมัครและยืนยันอีเมล GitHub ล่วงหน้า
- [ ] ทดสอบ browser-based Git authentication กับเครือข่ายสถานที่
- [ ] เตรียมไฟล์ starter และ asset สำรองผ่าน USB/local share เมื่ออินเทอร์เน็ตล่ม
- [ ] ยืนยันว่า starter ไม่มี `.git`, token, ข้อมูลจริง หรือเฉลย capstone
- [ ] เปิด W3C Validator และ DevTools responsive mode ไว้ใน bookmark
- [ ] พิมพ์/แชร์ [คู่มือแก้ปัญหา](/workshop/troubleshooting)

## ข้อตกลงก่อนเริ่ม

ให้ผู้เรียนยกมือเมื่อ checkpoint ไม่ผ่าน และไม่ปล่อยให้คัดลอกคำสั่งถัดไป เพราะ path ผิดหนึ่งครั้งจะทำให้ Git ทั้งช่วงทำงานคนละโฟลเดอร์ เน้นให้ “อ่าน output ก่อนพิมพ์คำสั่งถัดไป”

## Checkpoints ตามช่วงเวลา

| Stop point | หลักฐานที่ทุกคนต้องแสดง | ถ้าไม่ผ่าน |
|---|---|---|
| จบ Setup/History | เปิด starter ได้และตอบคำถามประวัติ 3 ใน 5 ข้อ | จับคู่กับเพื่อนและใช้คำตอบอ้างอิงด้านล่าง |
| ก่อนพักเช้า | `index.html` มี document skeleton, `lang`, title และเนื้อหาหลัก | แจก diff เฉพาะ block ที่ขาด ไม่แจกไฟล์เฉลยทั้งชุด |
| ก่อนพักกลางวัน | semantic regions, alt decision, validator และ keyboard pass | ย้าย table/form ไป enrichment |
| ก่อนเริ่ม Git | หน้า 320px ไม่ล้น, focus ชัด, `pwd` อยู่ `my-shop-profile` | ใช้ minimal CSS checkpoint แล้วตัด Grid |
| หลังช่วง Git/GitHub | `git log --oneline -1` มี baseline และ push แรกสำเร็จหรือมีหลักฐาน local | เก็บ baseline ในเครื่องแล้วใช้ network fallback |
| หลัง Capstone | `git log --oneline -3` มี baseline + milestone เนื้อหา + milestone responsive | กลับไปทำ checkpoint ที่ยังขาด ห้ามแก้ไฟล์เพื่อสร้างประวัติปลอม |
| ก่อนส่ง | rubric 4 หมวดผ่านและ `git status` clean | ส่ง local evidence ก่อนหาก auth/network ล่ม |

## คำถามประวัติพร้อมคำตอบอ้างอิง

1. **Web เกิดขึ้นเพื่อแก้ปัญหาอะไร?** — ช่วยให้นักวิทยาศาสตร์ต่างระบบแลกเปลี่ยนข้อมูลที่ CERN ได้สะดวกขึ้น
2. **ใครเขียน HTML รุ่นแรก และเมื่อใด?** — Tim Berners-Lee เขียนชุดแรกระหว่างสร้าง Web; browser/server แรกทำงานปลายปี 1990 หลังข้อเสนอปี 1989
3. **Internet กับ Web เหมือนกันหรือไม่?** — ไม่เหมือน; Internet คือเครือข่ายพื้นฐาน ส่วน Web เป็นระบบเอกสารเชื่อมโยงที่ทำงานบนเครือข่ายนั้น
4. **ทำไมต้องแยก HTML กับ CSS?** — HTML บอกโครงสร้าง/ความหมาย ส่วน CSS ควบคุมการนำเสนอ ทำให้เปลี่ยนรูปแบบได้โดยไม่ทำลายความหมายของเอกสาร
5. **ใครดูแลมาตรฐาน HTML ปัจจุบัน?** — WHATWG ดูแล HTML Living Standard โดย W3C และ WHATWG ตกลงทำมาตรฐานเดียวกันตั้งแต่ปี 2019

ให้คะแนนผ่านเมื่อผู้เรียนอธิบายด้วยภาษาตนเองได้ ไม่บังคับจำถ้อยคำหรือปีทุกปี แหล่งปฐมภูมิอยู่ใน [ประวัติ Web](/intro/web-history) และ [แหล่งเรียนรู้](/workshop/resources)

## สาขาสำหรับผู้เรียนที่ต้องการความช่วยเหลือ

- ตรวจ `pwd` และชื่อไฟล์ก่อนตรวจ syntax
- ให้ผู้เรียนอธิบายสิ่งที่คาดว่าจะเห็น แล้วเทียบ output จริงทีละบรรทัด
- จำกัดเป้าหมายที่ core: ตัด table, form, Grid และ clone ก่อน
- ใช้ไฟล์ checkpoint ล่าสุดที่ผ่าน แต่อย่าแทนที่ไฟล์ทั้งโปรเจกต์โดยไม่อธิบาย diff
- หาก GitHub มีปัญหา ให้รักษา local commits และรับ `git log` เป็นหลักฐานชั่วคราว

## สาขาสำหรับผู้เรียนที่เสร็จเร็ว

- ตรวจ alt ของเพื่อนด้วยคำถาม “ถ้าภาพไม่โหลด ยังรู้สาระสำคัญไหม”
- ทำ [ตาราง](/html/lists-tables), [ฟอร์ม](/html/forms), [CSS Grid](/css/grid) หรือ [`git clone`](/git/clone) ตามลำดับความสนใจ
- เพิ่ม README ที่บอกเป้าหมาย ภาพหน้าจอ และวิธีเปิดงาน แล้ว commit แยกหนึ่งเรื่อง
- ลองตรวจ 320/768/1280px และเขียน issue สั้น ๆ แทนการแก้ไฟล์ให้เพื่อน

## ก่อนปิดชั้นเรียน

- [ ] ผู้เรียนทุกคนรู้ว่างานอยู่ใน path ใดและตรวจด้วย `pwd` ได้
- [ ] ทุกคนมี local commit อย่างน้อย 3 จุด แม้ push ไม่สำเร็จ
- [ ] รับหลักฐานตาม rubric ไม่ให้คะแนนจากความสวยเพียงอย่างเดียว
- [ ] เตือนให้ลบ token ที่อาจเผลอเผยแพร่และ rotate ทันทีหากพบ
- [ ] ชี้เส้นทางเรียนต่อจาก [สรุป](/workshop/summary) และ [แหล่งเรียนรู้](/workshop/resources)
